# 改卷功能集成说明

## 已完成的功能

### 1. AI评分服务 (Python)
✅ **文件位置**：`springboot/src/main/resources/python/ai_grading_service.py`

**功能特点**：
- 基于BERT语义相似度（70%权重）
- TF-IDF文本相似度（20%权重）
- 关键词匹配度（10%权重）
- 长度惩罚机制
- 支持单题评分和批量评分

**API接口**：
- `GET /health` - 健康检查
- `POST /grade` - 单题评分
- `POST /grade/batch` - 批量评分

### 2. Java调用服务
✅ **PythonGradingService.java** - 封装Python API调用

**核心方法**：
- `healthCheck()` - 检查Python服务状态
- `gradeSingle(standard, student, maxScore)` - 单题评分
- `gradeBatch(answers)` - 批量评分

### 3. 改卷服务
✅ **ScoreService.java** - 完善改卷功能

**核心功能**：
- `selectAll(teacherId)` - 获取答卷列表
- `getScoreDetail(scoreId)` - 获取答卷详情
- `manualGrade(answerId, score, remark)` - 人工改卷
- `batchManualGrade(scoreId, scores)` - 批量改卷
- `confirmAiGrading(scoreId)` - 确认AI评分
- `getScoreStatistics(paperId)` - 成绩统计

### 4. 改卷API接口
✅ **ScoreController.java** - 提供改卷API

**API列表**：
- `GET /score/list?teacherId=xxx` - 获取答卷列表
- `GET /score/detail/{scoreId}` - 获取答卷详情
- `POST /score/grade/{answerId}` - 人工改卷（单题）
- `POST /score/batch/{scoreId}` - 批量改卷
- `POST /score/confirm-ai/{scoreId}` - 确认AI评分
- `GET /score/statistics?paperId=xxx` - 成绩统计

### 5. 测试接口
✅ **GradingTestController.java** - 测试AI评分

**API列表**：
- `GET /grading/health` - 检查Python服务
- `GET /grading/demo` - 快速测试评分示例
- `POST /grading/test` - 自定义测试评分

## 使用步骤

### 步骤1：启动Python服务

**方法一：使用启动脚本（Windows）**
```bash
cd springboot/src/main/resources/python
双击 start_ai_service.bat
```

**方法二：手动启动**
```bash
cd springboot/src/main/resources/python
pip install flask flask-cors sentence-transformers jieba scikit-learn scipy pandas numpy
python ai_grading_service.py
```

**首次启动**会自动下载BERT模型（约400MB），请耐心等待。

### 步骤2：启动Java后端

正常启动Spring Boot应用即可：
```bash
cd springboot
mvn spring-boot:run
```

### 步骤3：测试AI服务

**测试1：检查服务状态**
```
GET http://localhost:9090/grading/health
```

**测试2：快速测试评分**
```
GET http://localhost:9090/grading/demo
```

**测试3：自定义评分**

### 步骤4：使用改卷功能

**场景1：查看待改卷列表**
```

```

**场景2：查看答卷详情**
```
GET http://localhost:9090/score/detail/1
```

**场景3：人工改卷**

**场景4：确认AI评分**
```
POST http://localhost:9090/score/confirm-ai/1
```

## 数据库表结构补充

确保 `student_answer` 表包含以下字段：

```sql
CREATE TABLE IF NOT EXISTS `student_answer` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `student_id` int(11) NOT NULL,
  `paper_id` int(11) NOT NULL,
  `question_id` int(11) NOT NULL,
  `answer_text` text COMMENT '文本答案（填空、简答）',
  `answer_option` varchar(255) COMMENT '选项答案（选择题）',
  `max_score` int(11) COMMENT '该题满分',
  `ai_score` int(11) COMMENT 'AI评分',
  `final_score` int(11) COMMENT '最终分数（人工评分或AI评分）',
  `similarity` double COMMENT '相似度',
  `grading_status` varchar(20) COMMENT '评分状态：pending/ai_graded/graded',
  `remark` varchar(500) COMMENT '评语',
  `created_time` datetime DEFAULT CURRENT_TIMESTAMP,
  `updated_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_score` (`student_id`,`paper_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 评分流程说明

```
学生交卷
    ↓
客观题自动评分（单选、多选、判断）
    ↓
状态变为"grading"
    ↓
异步调用Python AI评分（填空、简答）
    ↓
AI评分完成，状态"ai_graded"
    ↓
教师查看答卷
    ↓
选择：
  ├─ 接受AI评分 → 点击"确认AI评分" → 状态"finished"
  └─ 人工修改 → 修改分数 → 状态"graded" → 可再次修改
```

```yaml
  service:
    url: http://localhost:5000
```

如果Python服务部署在其他机器上，修改此配置即可。

## 常见问题

### Q1: Python服务启动失败？

**检查**：
1. Python 3.8+ 是否安装
2. 依赖包是否安装完整
3. 端口5000是否被占用

### Q2: AI评分不准确？

**优化建议**：
1. 确保标准答案准确完整
2. 可以调整Python服务中的`score_thresholds`参数
3. 重要考试建议AI评分后人工复核

### Q3: 评分速度慢？

**优化**：
1. 首次启动需要下载模型，耐心等待
2. 批量评分比单题评分效率高
3. 确保服务器内存充足（至少4GB）

## 后续优化建议

1. **模型缓存**：标准答案的BERT向量可以缓存，避免重复计算
2. **异步优化**：大并发时考虑消息队列
3. **人工复核**：增加人工复核流程，提高评分准确性
4. **历史数据**：积累历史评分数据，训练更准确的评分模型
